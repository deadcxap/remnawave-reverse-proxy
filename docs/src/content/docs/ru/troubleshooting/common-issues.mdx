## Развертывание ботов

Если вы развертываете ботов, убедитесь, что в окружении или конфигурации бота вы используете режим `local`. В этом случае вы можете просто ввести `remnawave:3000`, чтобы избежать проблем с доступом к панели.

## Добавление зеркал для Docker

В некоторых случаях, как в [#40](https://github.com/eGamesAPI/remnawave-reverse-proxy/issues/40), серверы, особенно те, которые расположены в России, не могут загружать образы.
Чтобы решить эту проблему, изучите [документацию по добавлению зеркал для Docker](/ru/troubleshooting/docker-mirrors).

## You have reached your unauthenticated pull rate limit

Если вы видите это сообщение, это означает, что вы превысили лимит частоты для неаутентифицированных запросов к Docker Hub. Чтобы решить эту проблему, вы можете попробовать одно из следующих решений:

<Steps>

1. Авторизуйтесь в Docker Hub

    Войдите в свою учетную запись Docker Hub с помощью Docker CLI:

    ```bash
    docker login
    ```

2. Используйте другую сеть

    Если возможно, попробуйте переключиться на другую сеть, чтобы увидеть, сохраняется ли проблема. А еще лучше задать этот вопрос вашему хостингу.

3. Включите IPv6

    Запустите скрипт и выберите "7. Управление IPv6" и включите поддержку IPv6. Это может помочь обойти некоторые лимиты частоты в некоторых случаях.

</Steps>
